# セキュリティポリシー

## 概要

ウィークリータスクボードはクライアント側で動作するシンプルなタスク管理アプリケーションです。このドキュメントでは、セキュリティに関する考慮事項と推奨事項をまとめています。

## セキュリティの特徴

### 1. ローカルストレージのみ使用

- **データ保存**: すべてのタスクデータと設定は、ブラウザのローカルストレージにのみ保存されます
- **外部送信なし**: データは外部サーバーに送信されません
- **プライバシー**: ユーザーのデータは完全にローカルマシンに保存されます

### 2. クライアント側処理

- **サーバー不要**: バックエンドサーバーが不要なため、サーバー側の脆弱性がありません
- **オフライン動作**: インターネット接続がなくても動作します
- **スタンドアロン**: 外部APIやサービスに依存しません

### 3. データ検証

- **入力検証**: すべてのユーザー入力は検証されます
- **型チェック**: タスクデータの型が正しいことを確認します
- **範囲チェック**: 時間データなどの数値は妥当な範囲内であることを確認します

## セキュリティ上の考慮事項

### ローカルストレージのセキュリティ

#### リスク

1. **ブラウザアクセス**: 同じマシンにアクセスできるユーザーがローカルストレージを読み取ることができます
2. **XSS攻撃**: 悪意のあるスクリプトがローカルストレージにアクセスする可能性があります
3. **物理的アクセス**: マシンへの物理的アクセスがある場合、ローカルストレージが読み取られる可能性があります

#### 対策

1. **ブラウザセキュリティ**: 信頼できるブラウザを使用してください
2. **マシンセキュリティ**: マシンのロックとパスワード保護を有効にしてください
3. **ユーザー分離**: 複数ユーザーの環境では、各ユーザーが独立したブラウザプロファイルを使用してください

### XSS（クロスサイトスクリプティング）対策

#### 実装済みの対策

1. **入力サニタイズ**: ユーザー入力は適切に処理されます
2. **テキストノード**: HTMLコンテンツは`textContent`を使用して設定されます
3. **イベントハンドラ**: インラインイベントハンドラは使用されません

#### 推奨事項

1. **信頼できるソースのみ**: 信頼できるソースからのみこのアプリケーションを実行してください
2. **ブラウザ拡張機能**: 不明なブラウザ拡張機能をインストールしないでください
3. **定期的な更新**: ブラウザを最新の状態に保ってください

### データ暗号化

#### 現在の状態

- ローカルストレージのデータは暗号化されていません
- これはクライアント側アプリケーションの標準的な実装です

#### 推奨事項

1. **機密情報**: 機密情報をタスク詳細に含めないでください
2. **パスワード**: パスワードやAPIキーをタスクに保存しないでください
3. **個人情報**: 個人識別情報（PII）をタスクに含めないでください

## 脆弱性報告

### 報告方法

セキュリティ脆弱性を発見した場合は、以下の手順に従ってください：

#### 推奨: GitHub Security Advisory

**最も推奨される報告方法です。以下のリンクから直接申請してください：**

👉 **[GitHub Security Advisory - 脆弱性を報告](https://github.com/y-maeda1116/Weekly-Task-Board/security/advisories/new)**

このリンクから申請することで：
- 脆弱性情報が非公開で管理されます
- プロジェクト管理者に直接通知されます
- 修正までの間、情報が保護されます
- CVE番号の割り当てが可能になります

#### 代替方法

GitHub Security Advisoryが利用できない場合：

1. **公開しない**: GitHubのIssueで脆弱性を公開しないでください
2. **メール報告**: プロジェクト管理者にメールで報告してください
3. **詳細情報**: 脆弱性の詳細な説明と再現手順を含めてください

### 報告内容

脆弱性報告には以下の情報を含めてください：

- **脆弱性の説明**: 脆弱性の詳細な説明
- **影響を受けるコンポーネント**: 影響を受けるファイルやモジュール
- **再現手順**: 脆弱性を再現するための具体的な手順
- **潜在的な影響**: セキュリティへの潜在的な影響
- **推奨される修正方法**: 可能であれば修正方法を提案してください
- **CVSS スコア**: 可能であれば脆弱性の重大度を評価してください

### 報告後の流れ

1. **確認**: プロジェクト管理者が脆弱性を確認します
2. **評価**: 脆弱性の重大度と影響を評価します
3. **修正**: 脆弱性を修正するためのパッチを開発します
4. **テスト**: 修正が正しく機能することをテストします
5. **リリース**: セキュリティパッチをリリースします
6. **公開**: 脆弱性情報を公開します（修正後）

### 報告者への謝辞

セキュリティ脆弱性を報告していただいた方には、以下の方法で謝辞を表示します：

- GitHub Security Advisory の謝辞セクション
- リリースノート
- プロジェクトのセキュリティ貢献者リスト（希望する場合）

## 依存関係

### 外部ライブラリ

このプロジェクトは以下の外部ライブラリに依存しています：

- **なし**: このプロジェクトはバニラJavaScriptのみを使用しており、外部ライブラリに依存していません

### 利点

- 依存関係がないため、サプライチェーン攻撃のリスクがありません
- ライブラリの脆弱性の影響を受けません
- 更新の負担が少なくなります

## コンプライアンス

### GDPR（一般データ保護規則）

- **データ保存**: すべてのデータはローカルストレージに保存されます
- **データ削除**: ユーザーはブラウザのキャッシュをクリアすることでデータを削除できます
- **データ移植性**: ユーザーはエクスポート機能を使用してデータを移植できます

### その他の規制

- **CCPA（カリフォルニア消費者プライバシー法）**: 外部へのデータ送信がないため、CCPAの要件を満たしています
- **HIPAA（医療保険の携行と説明責任に関する法律）**: 医療データの保存には推奨されません

## よくある質問（FAQ）

### Q: データは暗号化されていますか？

A: いいえ、ローカルストレージのデータは暗号化されていません。これはクライアント側アプリケーションの標準的な実装です。機密情報をタスクに保存しないことをお勧めします。

### Q: データはどこに保存されていますか？

A: すべてのデータはブラウザのローカルストレージに保存されます。外部サーバーには送信されません。

### Q: 複数のデバイスでデータを同期できますか？

A: いいえ、このアプリケーションはクライアント側のみで動作するため、複数のデバイス間での自動同期はできません。ただし、エクスポート/インポート機能を使用してデータを手動で転送できます。

### Q: ブラウザのプライベートモードで使用できますか？

A: はい、使用できます。ただし、プライベートモードを終了するとローカルストレージのデータが削除される場合があります。

### Q: このアプリケーションはオフラインで動作しますか？

A: はい、完全にオフラインで動作します。インターネット接続は不要です。
