# Outlook カレンダー同期機能 - 設計ドキュメント

## 概要

Outlook カレンダー同期機能は、Microsoft Outlook カレンダーとタスクボード アプリケーション間のシームレスな統合を実現します。ユーザーは Outlook アカウントで認証し、指定された日付範囲からイベントを取得し、インポートするイベントを選択して、タスクボードにタスクとして追加できます。システムは同期状態を管理して重複インポートを防ぎ、自動再試行メカニズムを備えた堅牢なエラーハンドリングを提供します。

### 主要目標

- Outlook との安全な OAuth 認証を有効にする
- ユーザーが指定した日付範囲内のカレンダー イベントを取得する
- イベント選択とプレビューのための直感的な UI を提供する
- Outlook イベントをタスクボード タスクに変換し、すべてのメタデータを保持する
- 同期状態を追跡して重複を管理する
- 自動再試行を備えた回復力のあるエラーハンドリングを実装する
- 適切なシリアライゼーション/デシリアライゼーションを通じてデータ整合性を確保する

### コンポーネント相互作用フロー

1. **ユーザーが同期を開始** → カレンダー同期 UI
2. **UI が日付範囲を収集** → カレンダー インポーター
3. **カレンダー インポーターがイベントをリクエスト** → Outlook コネクター
4. **Outlook コネクターが認証** → Outlook API
5. **API がイベント データを返す** → イベント パーサー
6. **パーサーが Event オブジェクトに変換** → 同期エンジン
7. **同期エンジンが重複をチェック** → カレンダー インポーター
8. **ユーザーがイベントを選択** → カレンダー インポーター
9. **カレンダー インポーターがタスクに変換** → イベント シリアライザー
10. **タスクが永続化される** → タスクボード ストレージ

## コンポーネントとインターフェース

### 1. Outlook コネクター コンポーネント

**責務:** OAuth 認証と Outlook との API 通信を管理します。

**主要メソッド:**

```typescript
interface OutlookConnector {
  // 認証
  initiateOAuthFlow(): Promise<void>
  handleOAuthCallback(code: string): Promise<void>
  disconnectAccount(): Promise<void>
  isAuthenticated(): boolean
  
  // トークン管理
  getAccessToken(): string | null
  refreshAccessToken(): Promise<string>
  revokeAccessToken(): Promise<void>
  
  // イベント取得
  getEvents(startDate: Date, endDate: Date): Promise<RawEventData[]>
  getEventDetails(eventId: string): Promise<RawEventData>
}
```

**責務:**
- OAuth 2.0 認証フローを開始・処理する
- アクセス トークンを安全に保存・管理する
- 期限切れトークンを自動的に更新する
- Outlook Graph API と通信する
- API レート制限とエラーを処理する
- すべての認証と API 相互作用をログに記録する

**セキュリティ考慮事項:**
- トークンを安全な暗号化ストレージに保存する (localStorage ではない)
- トークン有効期限と更新ロジックを実装する
- すべての API 通信に HTTPS を使用する
- CSRF 攻撃を防ぐために OAuth state パラメーターを検証する

---

### 2. イベント パーサー コンポーネント

**責務:** 生の Outlook API レスポンスを構造化された Event オブジェクトに解析します。

**主要メソッド:**

```typescript
interface EventParser {
  parseEvent(rawData: RawEventData): Event
  parseEvents(rawDataArray: RawEventData[]): Event[]
  validateEventData(data: unknown): boolean
}
```

**データ変換:**
- Outlook API からの JSON を Event オブジェクトに変換する
- 必須フィールド (タイトル、開始、終了) を検証する
- タイムゾーン変換を処理する
- 日付/時刻形式を正規化する
- イベント メタデータを抽出・保持する

**エラーハンドリング:**
- 無効なデータに対して説明的なエラーをスローする
- パース失敗をコンテキスト付きでログに記録する
- オプション フィールドにフォールバック値を提供する

---

### 3. イベント シリアライザー コンポーネント

**責務:** Event オブジェクトをタスクボード用の Task オブジェクトに変換します。

**主要メソッド:**

```typescript

  eventToTask(event: Event): Task
  eventsToTasks(events: Event[]): Task[]
  taskToEvent(task: Task): Event
}
```

**変換ロジック:**
- Event フィールドを Task フィールドにマップする
- すべての関連メタデータを保持する
- 一意のタスク識別子を生成する
- イベント タイミングに基づいてタスク ステータスを設定する
- 同期追跡のための双方向マッピングを保持する

## データ モデル

### Event オブジェクト

```typescript
interface Event {
  id: string                    // Outlook イベント ID
  title: string                 // イベント タイトル
  description: string           // イベント説明
  startTime: Date              // イベント開始時刻 (UTC)
  endTime: Date                // イベント終了時刻 (UTC)
  location?: string            // イベント場所
  organizer?: string           // イベント主催者メール
  attendees?: string[]         // 出席者メール
  isAllDay: boolean            // 終日イベント フラグ
  recurrence?: RecurrenceRule  // 繰り返しパターン
  categories?: string[]        // イベント カテゴリ
  lastModified: Date           // 最終変更タイムスタンプ
  rawData?: Record<string, any> // 元の API レスポンス
}
```

### Task オブジェクト (参照用)

```typescript
interface Task {
  id: string                    // 一意のタスク ID
  title: string                 // タスク タイトル
  description: string           // タスク説明
  dueDate: Date                // タスク期限
  startDate?: Date             // タスク開始日
  endDate?: Date               // タスク終了日
  status: TaskStatus           // タスク ステータス (保留中、完了など)
  priority?: Priority          // タスク優先度
  tags?: string[]              // タスク タグ
  metadata?: {
    outlookEventId?: string    // 元の Outlook イベントへのリンク
    syncedAt?: Date            // 同期タイムスタンプ
    syncStatus?: SyncStatus    // 同期ステータス
  }
}
```

### 同期マッピング オブジェクト

```typescript
interface SyncMapping {
  id: string                    // マッピング ID
  outlookEventId: string       // Outlook イベント ID
  taskId: string               // タスクボード タスク ID
  syncedAt: Date               // 同期タイムスタンプ
  syncStatus: SyncStatus       // 現在の同期ステータス
  lastModified: Date           // 最終変更タイムスタンプ
}

enum SyncStatus {
  SYNCED = "synced",           // 同期済み
  PENDING = "pending",         // 保留中
  FAILED = "failed",           // 失敗
  DUPLICATE = "duplicate"      // 重複
}
```

## 正確性プロパティ

### プロパティ 1: OAuth フロー開始

**すべての** ユーザーが接続ボタンと相互作用するとき、システムは OAuth 2.0 認可フローを開始し、Microsoft ログイン ページにリダイレクトします。

---

### プロパティ 2: 安全なトークン ストレージ

**すべての** 成功した OAuth 認証に対して、システムはアクセス トークンを安全な暗号化ストレージに保存します (localStorage または sessionStorage ではない)。

---

### プロパティ 3: 切断時のトークン クリーンアップ

**すべての** 認証済みユーザーが切断をクリックするとき、システムはすべての保存されたトークンと認証状態を削除します。

---

### プロパティ 4: 日付範囲でのイベント取得

**すべての** 有効な日付範囲 (startDate ≤ endDate) に対して、システムは正しい日付パラメーターで Outlook API を呼び出し、その範囲内のイベントを返します。

---

### プロパティ 5: イベント データの完全性

**すべての** Outlook API から返されるイベントに対して、解析された Event オブジェクトはすべての必須フィールド (タイトル、startTime、endTime、説明) を含みます。

---

### プロパティ 6: 空のイベント リスト処理

**すべての** イベントがない日付範囲に対して、システムはエラーなしで空のリストを返します。

---

### プロパティ 7: イベント リスト表示

**すべての** Outlook から取得されたイベント セットに対して、カレンダー インポーター UI はすべてのイベントをチェックボックス付きのリスト形式で表示します。

---

### プロパティ 8: イベント詳細表示

**すべての** リストでクリックされたイベントに対して、システムはイベントのタイトル、開始時刻、終了時刻、説明を含む詳細パネルを表示します。

---

### プロパティ 9: 選択状態の永続化

**すべての** ユーザーが選択したイベントに対して、システムはユーザーが選択を解除するか、インポートを完了するまで選択状態を保持します。

---

### プロパティ 10: イベントからタスクへの変換

**すべての** 選択されたイベントに対して、システムはイベントのタイトルをタスク タイトルとして、イベント時刻をタスク日付として、イベント説明をタスク説明として使用して、Task オブジェクトに変換します。

---

### プロパティ 11: インポート トランザクション ロールバック

**すべての** エラーが発生したインポート操作に対して、システムはすべての変更をロールバックし、部分的なタスク レコードを作成せずにユーザーに通知します。

---

### プロパティ 12: 同期マッピング記録

**すべての** 正常にインポートされたイベントに対して、システムは将来の重複検出のために Outlook イベント ID と作成されたタスク ID 間のマッピングを記録します。

---

### プロパティ 13: 重複検出

**すべての** 以前にインポートされたイベントに対して、システムは再度インポートしようとするときそれを重複として検出します。

---

### プロパティ 14: 重複警告表示

**すべての** 検出された重複イベントに対して、システムはユーザーに警告を表示し、新しいタスクを作成するか、既存のタスクを更新するか、インポートをスキップするかのオプションを提供します。

---

### プロパティ 15: 日付範囲検証

**すべての** startDate > endDate である日付範囲に対して、システムはエラー メッセージを表示し、API 呼び出しを防ぎます。

---

### プロパティ 16: デフォルト日付範囲

**すべての** 明示的な日付範囲なしのインポート操作に対して、システムはデフォルトで本日のイベントを取得します。

---

### プロパティ 17: 指数バックオフを使用した再試行ロジック

**すべての** 操作中のネットワーク エラーに対して、システムは自動的に最大 3 回まで指数バックオフで再試行し、成功または最終的な失敗をユーザーに通知します。

---

### プロパティ 18: エラー ログ記録

**すべての** 同期操作中に発生するエラーに対して、システムはタイムスタンプ、エラー コード、コンポーネント名、関連コンテキストを含むエラーをログに記録します。

---

### プロパティ 19: イベント パース ラウンド トリップ

**すべての** Outlook API からの有効な JSON イベント データに対して、Event オブジェクトに解析し、Task オブジェクトにシリアライズし、解析し直すと、すべての元のデータが保持された同等の Event オブジェクトが生成されます。

---

### プロパティ 20: イベント フォーマット

**すべての** Event オブジェクトに対して、イベント プリンターはイベント タイトル、開始時刻、終了時刻、場所 (利用可能な場合) を含む人間が読める文字列にフォーマットします。

## テスト戦略

### デュアル テスト アプローチ

この機能には、包括的なカバレッジのためにユニット テストとプロパティベース テストの両方が必要です。

**ユニット テスト** (特定の例とエッジ ケース):
- モック OAuth レスポンスを使用した認証フロー
- 空のイベント リスト処理
- 無効な日付範囲検証
- インポート失敗時のトランザクション ロールバック
- さまざまなエラー シナリオのエラー メッセージ表示
- UI 相互作用テスト (ボタン クリック、チェックボックス選択)

**プロパティベース テスト** (ユニバーサル プロパティ):
- ランダムな日付範囲でのイベント取得
- すべての API レスポンス間でのイベント データ完全性
- ランダムなイベント選択での選択状態永続化
- ランダムなイベント データでのイベント-タスク変換
- ランダムなインポート シーケンスでの重複検出
- シミュレートされたネットワーク障害での再試行ロジック
- ランダムな JSON データでのイベント パース ラウンド トリップ
- ランダムなイベント オブジェクトでのイベント フォーマット

### テスト カバレッジ目標

- **ユニット テスト:** 最小 80% コード カバレッジ
- **プロパティ テスト:** すべての 20 プロパティを実装
- **統合テスト:** エンドツーエンド同期フロー
- **エラー シナリオ:** すべてのエラー パスをテスト
- **エッジ ケース:** 空のリスト、無効な日付、ネットワーク障害

## 結論

この設計は、Outlook カレンダーとタスクボード アプリケーション間の堅牢で安全、ユーザーフレンドリーな統合を提供します。アーキテクチャは関心の分離、包括的なエラーハンドリング、プロパティベースの正確性検証を強調しています。デュアル テスト アプローチにより、特定のエッジ ケースとユニバーサル プロパティの両方が検証され、信頼性の高い同期システムが実現します。
